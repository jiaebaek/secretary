# 주식비서(Secretary v2 REST) 사용 매뉴얼

> 코드 실제 동작 기준으로 작성. DB 항목 이름과 실제 동작이 다를 수 있으므로 **key 컬럼 문자열**과 **적용되는 전략**을 함께 확인하세요.

---

## 1. 시스템 개요

이 프로그램은 **키움 REST API**로 주문하고, **시간대별 전략**을 자동 실행하는 자동매매 시스템입니다.

| 구성요소 | 역할 |
|---------|------|
| `main.py` | 전략 1회 실행 (`--name` 또는 `--time`) |
| `scheduler.py` | `strategy_config.json` 시간표 실행 |
| `auto_scheduler.py` | `auto_strategy_config.json` 오토모드 반복 실행 |
| `strategy_ui.py` | Streamlit UI (전략/오토모드 설정) |
| `admin_panel.py` | Streamlit DB 설정 편집 |
| `D:\PycharmProjects\secretary_web\db.sqlite3` | 매매 파라미터 DB |

**매수는 코드상 3종류로 완전히 분리됩니다.**

| 종류 | 의미 | 제어 방법 |
|------|------|-----------|
| **현금 신규매수** | 관심종목 중 미보유 종목 매수 | `신규종목매수` DB값 (현재 스케줄에는 미사용) |
| **신용 신규매수** | 관심종목 중 미보유 종목 신용 매수 | `당일매수매도차이금액`, `신용매수설정` |
| **물타기(추가매수)** | 이미 보유 종목에 추가 매수 | `신용 물타기 기준(%)` 등 (신규매수 차단과 무관) |

## 2. 프로그램 실행 방법

`start_app.bat`은 **Streamlit UI + ngrok**만 띄웁니다. 매매 자동 실행을 하려면 아래도 별도로 실행해야 합니다.

```bat
python scheduler.py              :: 시간표 스케줄러
python auto_scheduler.py         :: 오토모드 스케줄러
python -m streamlit run strategy_ui.py   :: 전략 UI
python -m streamlit run admin_panel.py :: DB 설정 UI (원격)
```

- 전략 로그: `D:\secretary_log\`
- 스케줄러 로그: `logs\scheduler_YYYYMMDD.log`, `logs\auto_scheduler_YYYYMMDD.log`
- 실행 완료 시 텔레그램 알림 발송

## 3. 전략 실행 구조

### 3-1. 시간표 스케줄 (`strategy_config.json`)

특정 `HH:MM`에 등록된 전략을 **순서대로 1회** 실행합니다.

현재 주요 패턴 예:
- **07:59** — 매도 → 통합매수 → 대기(3분) → 미체결취소 → … (장 시작 전 루틴)
- **09:01~14:20** — `장중-신용-통합매수` (20분 간격)
- **15:33~** — 장마감 매도/취소 루틴

### 3-2. 오토모드 (`auto_strategy_config.json`)

현재 설정:
- **09:00~14:30** 구간에서 아래 묶음을 **성공할 때까지 무한 반복**

```
장중-신용-물타기-매수 → 대기(3분) → 미체결-현금신용매수-ALL-취소
→ 추가매수-미체결-매도-취소 → 장중-신용-매도
```

오토모드가 켜져 있으면(`auto_mode_enabled: true`), 같은 시간의 **시간표 `장중-신용-통합매수`는 스킵**됩니다.
대신 오토모드 묶음 안의 **물타기**가 반복 실행됩니다.

### 3-3. 거래소 자동 전환 (`set_exchange`)

| 시간 | 거래소 |
|------|--------|
| 08:35 이전 | NXT |
| 08:35 ~ 15:09 | KRX |
| 15:10 이후 | NXT |

## 4. 전략별 실제 동작 (등록된 전략)

### 매수 관련 (가장 중요)

| 전략명 | 실제 동작 |
|--------|-----------|
| **장중-신용-통합매수** | ① 신용 물타기(항상) → ② 조건 통과 시 신용 신규매수 |
| **장중-신용-물타기-매수** | 신용 물타기만 |
| **장중-신용-신규매수** | 신용 신규매수만 |
| **장중-현금-신규-매수** | 현금 신규매수 (`신규종목매수` 체크) — **현재 스케줄에 없음** |

### `장중-신용-통합매수` 상세 흐름

```
1. 잔고 조회 (신용 + 현금)
2. [물타기] 보유 종목 전체 순회
   - 수익률 ≤ 신용 물타기 기준(%) 이면 추가 매수
   - 현금+신용 합산 보유액 ≥ 종목별 매수 최대 금액 이면 스킵
3. [신규매수 차단 체크] 당일 (총매수 - 총매도) ≥ 당일매수매도차이금액 이면 신규매수 스킵
4. [신규매수] 키움 조건식 관심종목에서 미보유 종목 신용 매수
```

**물타기는 3번 차단과 무관하게 항상 실행됩니다.**

### 매도·취소·기타

| 전략명 | 동작 |
|--------|------|
| 장중-신용-매도 | 신용 잔고 단계별 지정가 매도 |
| 장중-현금-매도 | 현금 잔고 단계별 매도 |
| 장중-신용현금-매도 | 현금 + 신용 동시 매도 |
| 장마감-신용-매도 | 장마감 전 신용 정리 매도 |
| 시간외NXT-신용현금-매도 | 시간외 NXT 매도 |
| 미체결-현금신용매수-ALL-취소 | 미체결 매수 전부 취소 |
| 추가매수-미체결-매도-취소 | 물타기 후 꼬인 신용 매도 미체결 취소 |
| 대기 | 3분(180초) 대기 |
| 잔고-내역-저장 | DB `holding_summary` 갱신 |
| 매도주문-내역-삭제 | `local_order_history` 전체 삭제 |

## 5. DB 설정 — 이름 vs 실제 동작

DB는 `secretary_setup` 테이블의 **`key` 컬럼 문자열**과 매칭됩니다. (`id`, `value`, `key` 구조)

### 5-1. `secretary_setup` (기본설정)

| DB key (정확히 이 문자열) | 코드 변수 | 실제 동작 |
|---------------------------|----------|-----------|
| **신규종목매수** | `new_buy_stock` | **현금 신규매수 전략에서만** 사용. `0`이면 현금 신규매수 스킵. **신용 신규매수·물타기에는 영향 없음** |
| **운영상태** | `running_state` | DB에서 읽기만 하고 **코드 어디에서도 사용하지 않음** → 바꿔도 매매에 영향 없음 |
| **당일매수매도차이금액** | `max_trading_diff` | **신용 신규매수만** 제한. 조건: `(당일 총매수 - 당일 총매도) >= 이 값`이면 신규매수 차단 |
| **신규매수주식금액** | `buy_new_stock_amount` | **현금** 신규매수 1건 금액. `secretary_setup`에만 있음 |
| **기본 보유종목 개수** | `default_buy_new_stock_num` | 로드되지만 `set_buy_stock_num()`에서 **사용 안 함**(코드 주석 처리) |
| **종목별 매수 최대 금액(원)** | `max_amount` | 현금 물타기 한도 + **신용 물타기 시 현금+신용 합산 한도** |
| **신용 물타기 기준(%)** | `rebuy_credit_earning_rate` | 이 수익률 이하일 때 물타기 매수 시도 |
| **신용 물타기 매수금액** | `rebuy_credit_stock_amount` | 물타기 1회 매수 금액 |
| **신용 물타기 최대금액** | `credit_max_amount` | 신용 물타기 종목별 상한 |
| **신용 물타기 제외 종목** | `except_credit_rebuy_list` | `/`로 구분. 종목명 **완전 일치** 시 제외 (`*삼성전자` 형태) |
| **물타기 제외 종목** | `except_rebuy_list` | 현금 물타기 제외 (현재 스케줄에서 현금 물타기 전략 미사용) |
| **주문간격(초)** | `order_interval` | 주문 간 대기(로드만, 일부 경로에서 참조) |

### 5-2. `신용매수설정` 테이블 (별도!)

| key | 실제 동작 |
|-----|-----------|
| **신규매수주식개수** | 신용 신규매수 시 관심종목에서 최대 시도 종목 수. **`0`이면 신규매수 루프 즉시 종료** |
| **신규매수주식금액** | 신용 신규매수 1건 금액. **`0`이면 1주 매수** (금액 0 = 차단이 아님) |

### 5-3. 이름이 같아 헷갈리는 항목

| 항목 | 위치 | 영향 범위 |
|------|------|-----------|
| 신규매수주식금액 | `secretary_setup` | **현금** 신규매수 |
| 신규매수주식금액 | `신용매수설정` | **신용** 신규매수 |
| 신규매수주식개수 | `신용매수설정`만 존재 | **신용** 신규매수 (`secretary_setup`에는 없음) |

## 6. 신규매수 차단 — 올바른 방법

### 6-1. 신용 **신규매수**만 막기 (물타기는 유지)

**방법 A (권장): 스케줄 변경**
- `장중-신용-통합매수` → `장중-신용-물타기-매수`로 교체
- 신규매수 단계가 아예 실행되지 않음

**방법 B: DB 설정**
- `신용매수설정` → **신규매수주식개수 = 0**
- 또는 `당일매수매도차이금액`을 **매우 작은 음수** (예: `-999999999`)
  → 당일 차이가 거의 항상 기준 이상이 되어 신규매수 차단

**방법 C: 오토모드**
- 오토모드 묶음에 `장중-신용-통합매수`가 없으면 OK (현재는 물타기만 있음)

### 6-2. `당일매수매도차이금액` — 자주 틀리는 부분

조건식:

```
당일 (총매수금액 - 총매도금액) >= 당일매수매도차이금액  →  신용 신규매수 차단
```

| 설정값 | 의미 |
|--------|------|
| **0** | 당일 순매수가 0원 이상이면 차단. **장 초(매도가 더 많을 때)에는 신규매수 허용** |
| **30,000,000** (기본값) | 순매수 3천만 원 넘을 때만 차단 |
| **-999,999,999** | 거의 항상 차단 |

**실제 로그 사례 (2026-06-12):**
- 08:13 — 차이 `-4,843,960원`, 기준 `0원` → **신규매수 실행됨** (3건)
- 09:05 이후 — 차이 `+7,000,000원` 이상, 기준 `0원` → **신규매수 차단됨**
- 이후 물타기는 계속 실행됨

`0`을 "매수 금지"로 이해하면 **장 초에 신규매수가 나갈 수 있습니다.**

### 6-3. `신규종목매수`만 0으로 바꾼 경우

- **신용 신규매수에는 전혀 영향 없음**
- 현재 활성 스케줄에 `장중-현금-신규-매수`가 없어서 **실질 효과도 없음**

### 6-4. `운영상태`를 0으로 바꾼 경우

- 코드에서 **참조하지 않음** → 아무 일도 안 일어남

### 6-5. 물타기까지 막기

DB만으로는 불완전합니다. 아래를 함께 쓰세요.

1. 스케줄/오토모드에서 `장중-신용-물타기-매수`, `장중-신용-통합매수` 제거
2. 또는 `신용 물타기 기준(%)`을 `-99` 등 극단적으로 낮춤 (거의 모든 종목이 기준 미달)
3. 또는 `신용 물타기 제외 종목`에 제외할 종목명을 `/`로 나열

## 7. 사용자 실수 사례 정리

| 시도한 설정 | 기대 | 실제 |
|------------|------|------|
| `신규종목매수 = 0` | 모든 신규매수 차단 | **신용 신규매수·물타기 무관**. 현금 신규 전략도 스케줄에 없음 |
| `당일매수매도차이금액 = 0` | 매수 완전 차단 | **순매수가 음수일 때(장 초) 신규매수 허용**. 08:13에 3건 체결 |
| `신용매수설정` 금액/개수 = 0 | 매수 차단 | 개수 0은 신규 차단 OK. **금액 0은 1주 매수** |
| `운영상태 = 0` | 프로그램 정지 | **코드 미사용, 무효** |
| DB 수정 후에도 매수됨 | 즉시 반영 | 전략은 **실행 시작 시** `update_options()`로 DB 읽음. **이미 돌고 있는 프로세스에는 미반영** |

## 8. 신규매수 차단 체크리스트

신용 신규매수를 확실히 막으려면:

- [ ] `strategy_config.json` / 오토모드에서 `장중-신용-통합매수`, `장중-신용-신규매수` 제거 또는 `장중-신용-물타기-매수`만 사용
- [ ] `신용매수설정` → **신규매수주식개수 = 0**
- [ ] `당일매수매도차이금액` = **-999999999** (또는 충분히 작은 음수)
- [ ] `신규종목매수`는 신용에 영향 없음 — 혼동 금지
- [ ] DB 수정 후 **다음 전략 실행 전**에 저장됐는지 확인
- [ ] 로그에서 `!!! 신규 매수 제한` 또는 `신용주식 신규 매수 진행` 문구 확인

물타기까지 막으려면 추가로:

- [ ] 스케줄에서 물타기/통합매수 전략 제거
- [ ] 또는 `신용 물타기 기준(%)` 극단 조정 / 제외 종목 등록

## 9. 로그로 동작 확인하는 법

| 로그 문구 | 의미 |
|-----------|------|
| `>>>>>>>>>>> 신용주식 추가 매수 (물타기)` | 물타기 시작 (신규 차단과 무관) |
| `매수-매도 차이 체크: X원 (기준: Y원)` | 신용 신규매수 차단 판단 직전 |
| `!!! 신규 매수 제한` | 신용 신규매수 **차단됨** |
| `매수 조건 충족: 신용주식 신규 매수 진행` | 신용 신규매수 **실행됨** |
| `------- 지정가로 매수!!` (관심종목) | 신용 **신규** 매수 주문 |
| `------- 신용 현재가 매수!!` (보유종목) | **물타기** 매수 주문 |
| `>>>>>>>>>>>> 현금주식 신규 매수` | 현금 신규매수 (`신규종목매수` 체크) |

## 10. 핵심 요약

1. **이름에 "신규"가 있어도 전부 같은 스위치가 아닙니다.** `신규종목매수`는 현금 전용, 신용은 `신용매수설정`과 `당일매수매도차이금액`이 별도입니다.
2. **`장중-신용-통합매수`는 물타기를 먼저 하고**, 그다음 신규매수 여부를 판단합니다. DB로 신규만 막아도 **물타기는 계속**됩니다.
3. **`당일매수매도차이금액 = 0`은 "완전 차단"이 아닙니다.** 장 초 순매도 상태에서는 신규매수가 나갈 수 있습니다.
4. **`운영상태`는 현재 무효**합니다. 스케줄/오토모드 on/off로 제어하세요.
5. 설정 변경은 **다음 `main.py` 실행 시** 반영됩니다. 실행 중인 프로세스에는 적용되지 않습니다.
