코드 실제 동작 기준으로 작성. DB 항목 이름과 실제 동작이 다를 수 있으므로 key 컬럼 문자열과 적용되는 전략을 함께 확인하세요.

1. 시스템 개요

이 프로그램은 키움 REST API로 주문하고, 시간대별 전략을 자동 실행하는 자동매매 시스템입니다.

구성요소역할
main.py전략 1회 실행 (--name 또는 --time)
scheduler.pystrategy_config.json 시간표 실행
auto_scheduler.pyauto_strategy_config.json 오토모드 반복 실행
strategy_ui.pyStreamlit UI (전략/오토모드 설정)
admin_panel.pyStreamlit DB 설정 편집
D:\PycharmProjects\secretary_web\db.sqlite3매매 파라미터 DB

매수는 코드상 3종류로 완전히 분리됩니다.

종류의미제어 방법
현금 신규매수관심종목 중 미보유 종목 매수신규종목매수 DB값 (현재 스케줄에는 미사용)
신용 신규매수관심종목 중 미보유 종목 신용 매수당일매수매도차이금액, 신용매수설정
물타기(추가매수)이미 보유 종목에 추가 매수신용 물타기 기준(%) 등 (신규매수 차단과 무관)

2. 프로그램 실행 방법

start_app.bat은 Streamlit UI + ngrok만 띄웁니다. 매매 자동 실행을 하려면 아래도 별도로 실행해야 합니다.

python scheduler.py              :: 시간표 스케줄러
python auto_scheduler.py         :: 오토모드 스케줄러
python -m streamlit run strategy_ui.py   :: 전략 UI
python -m streamlit run admin_panel.py :: DB 설정 UI (원격)
  • 전략 로그: D:\secretary_log\
  • 스케줄러 로그: logs\scheduler_YYYYMMDD.log, logs\auto_scheduler_YYYYMMDD.log
  • 실행 완료 시 텔레그램 알림 발송

3. 전략 실행 구조

3-1. 시간표 스케줄 (strategy_config.json)

특정 HH:MM에 등록된 전략을 순서대로 1회 실행합니다.

  • 07:59 — 매도 → 통합매수 → 대기(3분) → 미체결취소 → … (장 시작 전 루틴)
  • 09:01~14:20 — 장중-신용-통합매수 (20분 간격)
  • 15:33~ — 장마감 매도/취소 루틴

3-2. 오토모드 (auto_strategy_config.json)

현재 설정: 09:00~14:30 구간에서 아래 묶음을 성공할 때까지 무한 반복

장중-신용-물타기-매수 → 대기(3분) → 미체결-현금신용매수-ALL-취소
→ 추가매수-미체결-매도-취소 → 장중-신용-매도

오토모드가 켜져 있으면(auto_mode_enabled: true), 같은 시간의 시간표 장중-신용-통합매수는 스킵됩니다. 대신 오토모드 묶음 안의 물타기가 반복 실행됩니다.

4. 전략별 실제 동작 (등록된 전략)

매수 관련 (가장 중요)

전략명실제 동작
장중-신용-통합매수① 신용 물타기(항상) → ② 조건 통과 시 신용 신규매수
장중-신용-물타기-매수신용 물타기만
장중-신용-신규매수신용 신규매수만
장중-현금-신규-매수현금 신규매수 (신규종목매수 체크) — 현재 스케줄에 없음

장중-신용-통합매수 상세 흐름

1. 잔고 조회 (신용 + 현금)
2. [물타기] 보유 종목 전체 순회
   - 수익률 ≤ 신용 물타기 기준(%) 이면 추가 매수
   - 현금+신용 합산 보유액 ≥ 종목별 매수 최대 금액 이면 스킵
3. [신규매수 차단 체크] 당일 (총매수 - 총매도) ≥ 당일매수매도차이금액 이면 신규매수 스킵
4. [신규매수] 키움 조건식 관심종목에서 미보유 종목 신용 매수
물타기는 3번 차단과 무관하게 항상 실행됩니다.

5. DB 설정 — 이름 vs 실제 동작

DB는 secretary_setup 테이블의 key 컬럼 문자열과 매칭됩니다. (id, value, key 구조)

6. 신규매수 차단 — 올바른 방법

6-1. 신용 신규매수만 막기 (물타기는 유지)

방법 A (권장): 스케줄 변경

  • 장중-신용-통합매수 → 장중-신용-물타기-매수로 교체
  • 신규매수 단계가 아예 실행되지 않음

방법 B: DB 설정

  • 신용매수설정 → 신규매수주식개수 = 0
  • 또는 당일매수매도차이금액을 매우 작은 음수 (예: -999999999) → 당일 차이가 거의 항상 기준 이상이 되어 신규매수 차단

방법 C: 오토모드

  • 오토모드 묶음에 장중-신용-통합매수가 없으면 OK (현재는 물타기만 있음)

6-2. 당일매수매도차이금액 — 자주 틀리는 부분

당일 (총매수금액 - 총매도금액) >= 당일매수매도차이금액  →  신용 신규매수 차단
설정값의미
0당일 순매수가 0원 이상이면 차단. 장 초(매도가 더 많을 때)에는 신규매수 허용
30,000,000 (기본값)순매수 3천만 원 넘을 때만 차단
-999,999,999거의 항상 차단
실제 로그 사례 (2026-06-12):
  • 08:13 — 차이 -4,843,960원, 기준 0원 → 신규매수 실행됨 (3건)
  • 09:05 이후 — 차이 +7,000,000원 이상, 기준 0원 → 신규매수 차단됨
  • 이후 물타기는 계속 실행됨

0을 "매수 금지"로 이해하면 장 초에 신규매수가 나갈 수 있습니다.

6-3. 신규종목매수만 0으로 바꾼 경우

  • 신용 신규매수에는 전혀 영향 없음
  • 현재 활성 스케줄에 장중-현금-신규-매수가 없어서 실질 효과도 없음

6-4. 운영상태를 0으로 바꾼 경우

  • 코드에서 참조하지 않음 → 아무 일도 안 일어남

6-5. 물타기까지 막기

DB만으로는 불완전합니다. 아래를 함께 쓰세요.

  1. 스케줄/오토모드에서 장중-신용-물타기-매수, 장중-신용-통합매수 제거
  2. 또는 신용 물타기 기준(%)을 -99 등 극단적으로 낮춤
  3. 또는 신용 물타기 제외 종목에 제외할 종목명을 /로 나열

7. 사용자 실수 사례 정리

시도한 설정기대실제
신규종목매수 = 0모든 신규매수 차단신용 신규매수·물타기 무관. 현금 신규 전략도 스케줄에 없음
당일매수매도차이금액 = 0매수 완전 차단순매수가 음수일 때(장 초) 신규매수 허용. 08:13에 3건 체결
신용매수설정 금액/개수 = 0매수 차단개수 0은 신규 차단 OK. 금액 0은 1주 매수
운영상태 = 0프로그램 정지코드 미사용, 무효
DB 수정 후에도 매수됨즉시 반영전략은 실행 시작 시 update_options()로 DB 읽음. 이미 돌고 있는 프로세스에는 미반영

8. 신규매수 차단 체크리스트

신용 신규매수를 확실히 막으려면:

  • strategy_config.json / 오토모드에서 장중-신용-통합매수, 장중-신용-신규매수 제거 또는 장중-신용-물타기-매수만 사용
  • 신용매수설정 → 신규매수주식개수 = 0
  • 당일매수매도차이금액 = -999999999 (또는 충분히 작은 음수)
  • 신규종목매수는 신용에 영향 없음 — 혼동 금지
  • DB 수정 후 다음 전략 실행 전에 저장됐는지 확인
  • 로그에서 !!! 신규 매수 제한 또는 신용주식 신규 매수 진행 문구 확인

물타기까지 막으려면 추가로:

  • 스케줄에서 물타기/통합매수 전략 제거
  • 또는 신용 물타기 기준(%) 극단 조정 / 제외 종목 등록

10. 핵심 요약

  1. 이름에 "신규"가 있어도 전부 같은 스위치가 아닙니다. 신규종목매수는 현금 전용, 신용은 신용매수설정과 당일매수매도차이금액이 별도입니다.
  2. 장중-신용-통합매수는 물타기를 먼저 하고, 그다음 신규매수 여부를 판단합니다. DB로 신규만 막아도 물타기는 계속됩니다.
  3. 당일매수매도차이금액 = 0은 "완전 차단"이 아닙니다. 장 초 순매도 상태에서는 신규매수가 나갈 수 있습니다.
  4. 운영상태는 현재 무효합니다. 스케줄/오토모드 on/off로 제어하세요.
  5. 설정 변경은 다음 main.py 실행 시 반영됩니다. 실행 중인 프로세스에는 적용되지 않습니다.
